Service Exchange API Documentation

RESTful API for the Service Exchange marketplace with seat-based verification

Base URL: https://rse-api.com:5003
Quick Navigation

Authentication

All API endpoints (except /ping, /register, and /login) require authentication via Bearer token.

Authorization: Bearer your_access_token

Tokens are obtained by calling /login and expire after 24 hours.

Seat Verification System

⚠️ Important: Provider Access Control

The /grab_job endpoint requires valid seat credentials to prevent marketplace abuse and ensure quality service providers.

Seat Types

GOLDEN Permanent Seats

Golden seats provide permanent access to the job matching system. These seats never expire and are issued to established service providers.

SILVER Time-Limited Seats

Silver seats provide access for one year from the assignment date. After expiration, providers must renew their seat credentials.

Seat Credential Structure

Field Type Description
seat.id string Unique seat identifier (e.g., "seat_abc123")
seat.owner string Registered owner of the seat
seat.secret string MD5 hash of your seat phrase (never send the phrase directly)

How to Generate Your Secret

Your seat secret must be the MD5 hash of your seat phrase. Never send the raw phrase.

// JavaScript Example const crypto = require('crypto'); const secret = crypto.createHash('md5').update('your-seat-phrase').digest('hex'); // Python Example import hashlib secret = hashlib.md5('your-seat-phrase'.encode()).hexdigest() // Command Line Example echo -n "your-seat-phrase" | md5sum

Verification Errors

Error Message Reason Solution
Missing seat ID No seat credentials provided Include seat object in request
Invalid golden/silver seat credentials Wrong owner or secret Verify your credentials match issued seat
Silver seat has expired One year period exceeded Contact admin for seat renewal
Seat ID not found Seat doesn't exist Verify seat ID is correct

POST /register

Create a new user account.

Request Body

Parameter Type Description
username string Unique username (3-20 characters)
password string Password (minimum 8 characters)

Example Request

curl -X POST https://rse-api.com:5003/register \ -H "Content-Type: application/json" \ -d '{"username": "john_doe", "password": "securepass123"}'

Example Response

{ "message": "Registration successful" }

POST /submit_bid

Submit a service request as a buyer. Requires authentication.

Request Body

Parameter Type Description
service string Description of service needed
price number Price willing to pay (USD)
end_time integer Unix timestamp when bid expires
location_type string "physical", "remote", or "hybrid"
address string Address (for physical services)
lat number Latitude (alternative to address)
lon number Longitude (alternative to address)

Example Request

curl -X POST https://rse-api.com:5003/submit_bid \ -H "Authorization: Bearer your_access_token" \ -H "Content-Type: application/json" \ -d '{ "service": "House cleaning, 3 bedrooms, 2 bathrooms", "price": 150, "end_time": 1703088000, "location_type": "physical", "address": "123 Main St, Denver, CO 80202" }'

Example Response

{ "bid_id": "b1234567-89ab-cdef-0123-456789abcdef" }

POST /grab_job SEAT REQUIRED

⚠️ Seat Verification Required

This endpoint requires valid seat credentials. You must include your seat information in the request body to access the job matching system.

Get matched with the best available job based on capabilities and location. Requires both authentication AND valid seat credentials.

Matching Algorithm Priority

  1. Seat Verification - Valid golden or silver seat required
  2. Location Filtering - Jobs within your specified distance
  3. Capability Matching - Jobs that match your skills
  4. Reputation Ranking - Better reputation alignment
  5. Price Ranking - Highest paying within same reputation tier

Request Body

Parameter Type Required Description
seat object Yes Seat verification credentials (see structure below)
seat.id string Yes Your unique seat identifier
seat.owner string Yes Registered owner name for the seat
seat.secret string Yes MD5 hash of your seat phrase
capabilities string Yes Your skills and services offered
location_type string No "physical", "remote", or "hybrid"
address string Conditional Your location (required for physical services)
lat number No Latitude (alternative to address)
lon number No Longitude (alternative to address)
max_distance number No Maximum travel distance in miles (default: 10)

Example Request with Seat Credentials

curl -X POST https://rse-api.com:5003/grab_job \ -H "Authorization: Bearer your_access_token" \ -H "Content-Type: application/json" \ -d '{ "seat": { "id": "seat_abc123", "owner": "john_doe", "secret": "5d41402abc4b2a76b9719d911017c592" }, "capabilities": "House cleaning, office cleaning, deep cleaning", "location_type": "physical", "address": "456 Oak Ave, Denver, CO 80203", "max_distance": 15 }'

Example Success Response

{ "job_id": "j9876543-21ba-dcfe-3210-fedcba987654", "bid_id": "b1234567-89ab-cdef-0123-456789abcdef", "status": "accepted", "service": "House cleaning, 3 bedrooms, 2 bathrooms", "price": 150, "location_type": "physical", "address": "123 Main St, Denver, CO 80202", "buyer_username": "jane_smith", "provider_username": "john_doe", "accepted_at": 1703002000, "buyer_reputation": 4.3, "provider_reputation": 4.2 }

Example Seat Verification Error

{ "error": "Seat verification failed: Invalid golden seat credentials" }

Error Responses

All errors follow a consistent format:

{ "error": "Error message describing what went wrong" }

Common HTTP Status Codes

Code Description
200 Success
201 Created (successful registration)
204 No Content (no matching jobs available)
400 Bad Request (invalid parameters)
401 Unauthorized (invalid or missing token)
403 Forbidden (not authorized for this action, includes seat verification failures)
404 Not Found
500 Internal Server Error

Rate Limiting

API requests are limited to 60 requests per minute per IP address. Exceeding this limit will result in a 429 Too Many Requests response.

Obtaining Seats

Seats are issued by the Service Exchange administrators to verified service providers. To request a seat:

  1. Complete your provider profile with verified credentials
  2. Maintain a reputation score of 3.5 or higher
  3. Complete at least 5 jobs successfully
  4. Contact support@rse-api.com with your provider details

Note: Seat credentials are confidential. Never share your seat phrase or secret with others. Compromised seats will be revoked.
